## 1. 改版背景

### 1.1 当前问题

| 问题 | 描述 |
|------|------|
| 地图加载影响首屏速度 | 首页打开需初始化地图SDK，加载地图资源，影响首屏渲染速度 |
| 新增服务无展示入口 | 卖车、金融贷款、租电等新业务上线，首页缺少入口位置 |
| 首页定位单一 | 当前首页仅聚焦换电功能，无法承载多业务聚合 |

### 1.2 改版目标

| 目标 | 描述 |
|------|------|
| 提升首屏加载速度 | 地图延迟加载，减少首页资源请求 |
| 服务聚合展示 | 提供多业务入口，支持服务扩展 |
| 简化信息展示 | 用户状态精简，降低视觉干扰 |

## 2. 信息架构

### 2.1 改版前

```
首页
├── 用户套餐卡片（顶部，占位较大）
├── 地图换电区（主体，首次加载即初始化地图）
└── 底部导航
```

### 2.2 改版后

```
首页
├── 顶部导航栏（融入用户状态）
├── 服务宫格入口（4个：换电、卖车、金融贷款、租电）
├── 推荐卡片轮播区
│   ├── 活动/优惠卡片
│   └── 附近换电站卡片
└── 底部导航
```

## 3. 页面布局设计

### 3.1 整体布局

```
┌─────────────────────────────────────────┐
│ Logo      页面标题      余额 | 套餐状态  │  ← 顶部导航栏（含状态）
├─────────────────────────────────────────┤
│                                         │
│  ┌─────────┐  ┌─────────┐              │
│  │  换电   │  │  卖车   │              │
│  └─────────┘  └─────────┘              │  ← 服务宫格（2×2）
│  ┌─────────┐  ┌─────────┐              │
│  │金融贷款 │  │  租电   │              │
│  └─────────┘  └─────────┘              │
│                                         │
├─────────────────────────────────────────┤
│  ← 推荐卡片轮播 →                        │
│  ┌─────────────────────────────────────┐│
│  │ 活动卡片：套餐促销/优惠活动           ││  ← 运营活动
│  └─────────────────────────────────────┘│
│  ┌─────────────────────────────────────┐│
│  │ 附近换电站：站名 | 距离 | 可换电池    ││  ← 站点信息
│  └─────────────────────────────────────┘│
└─────────────────────────────────────────┘
```

### 3.2 顶部导航栏

**设计说明：**
- 左侧：品牌Logo
- 中间：页面标题（默认"首页"）
- 右侧：用户状态信息

**状态信息元素：**

| 元素 | 展示内容 | 交互行为 |
|------|----------|----------|
| 余额 | 当前余额金额 | 点击跳转充值页 |
| 套餐状态 | 套餐名称或剩余天数 | 点击展开套餐详情弹窗 |

### 3.3 服务宫格入口

**布局：** 2×2 四宫格，同级平铺

| 位置 | 服务名称 | 交互行为 |
|------|----------|----------|
| 左上 | 换电 | 跳转独立地图换电页面 |
| 右上 | 卖车 | 跳转卖车服务落地页 |
| 左下 | 金融贷款 | 跳转金融贷款落地页 |
| 右下 | 租电 | 跳转租电服务落地页 |

**设计规范：**
- 图标风格统一，遵循现有设计语言
- 图标下方配文字标签
- 点击区域足够大，易于触控

### 3.4 推荐卡片轮播区

**内容类型：**

#### 3.4.1 运营活动卡片

| 字段 | 说明 |
|------|------|
| 活动标题 | 主标题文案 |
| 活动副标题 | 补充说明（可选） |
| 活动图片 | 卡片背景图/主图 |
| 标签 | 如"限时"、"热门"等 |

**交互：** 点击跳转活动详情页

#### 3.4.2 附近换电站卡片

| 字段 | 说明 |
|------|------|
| 站点名称 | 换电站名称 |
| 距离 | 距当前定位距离 |
| 可换电池数 | 当前可换电池数量 |
| 营业状态 | 营业中/休息中 |

**交互：** 点击跳转站点详情或地图换电页

**展示规则：**
- 展示最近1-2个换电站
- 按距离排序

## 4. 交互流程

### 4.1 首页加载流程

```
用户打开首页
    │
    ├── 加载顶部导航栏
    │   └── 请求用户状态接口（余额、套餐）
    │
    ├── 渲染服务宫格（静态，无需网络请求）
    │
    ├── 加载推荐卡片
    │   ├── 并行请求活动配置接口
    │   └── 并行请求附近换电站接口
    │
    └── 首屏渲染完成
```

**性能收益：**
- 无地图SDK初始化
- 无地图瓦片加载
- 首屏接口精简

### 4.2 换电流程

```
用户点击"换电"入口
    │
    └── 跳转地图换电页面
        │
        ├── 初始化地图SDK
        ├── 加载地图瓦片
        ├── 请求换电站列表数据
        │
        └── 展示地图换电界面
```

**设计理由：**
- 换电是独立功能模块，跳转页面符合用户预期
- 地图资源仅在需要时加载，避免资源浪费

## 5. 技术实现要点

### 5.1 性能优化方案

| 优化点 | 实现方案 | 预期收益 |
|--------|----------|----------|
| 地图延迟加载 | 仅在进入换电页时初始化地图SDK | 首屏加载时间减少1-2秒 |
| 接口并行请求 | 用户状态、活动、附近站点接口并行 | 减少请求等待时间 |
| 图片懒加载 | 卡片图片按需加载 | 减少首屏资源体积 |

### 5.2 接口需求

| 接口名称 | 用途 | 请求参数 | 返回字段 |
|----------|------|----------|----------|
| 用户状态接口 | 获取余额、套餐信息 | userId | balance, packageName, packageDays |
| 活动配置接口 | 获取当前活动列表 | city | activities[] |
| 附近换电站接口 | 获取最近换电站 | lat, lng, limit=2 | stations[] |

### 5.3 兼容性要求

| 平台 | 要求 |
|------|------|
| Android | 5.0+ |
| iOS | 12.0+ |
| 屏幕适配 | 支持主流屏幕尺寸 |

## 6. 改版前后对比

| 维度 | 改版前 | 改版后 |
|------|--------|--------|
| 首屏加载 | 需初始化地图SDK | 无地图加载 |
| 服务入口 | 仅换电 | 4个入口（换电、卖车、金融、租电） |
| 用户状态 | 顶部卡片，占位较大 | 融入导航栏，一行展示 |
| 信息密度 | 地图主导，信息单一 | 卡片混合，信息丰富 |
| 扩展性 | 新增服务无处放置 | 宫格可扩展，卡片可动态配置 |
